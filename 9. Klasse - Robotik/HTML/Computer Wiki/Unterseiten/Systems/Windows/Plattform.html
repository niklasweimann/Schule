Plattform

Das Konzept von Microsoft Windows ist es, eine möglichst binärkompatible Plattform für Anwendungsprogramme zu sein. Dies wird über die Stabile Win32-API realisiert. Seit den Anfängen ist Windows eine offene Plattform, heißt, jeder kann
uneingeschränkt Anwendungen für Windows schreiben und vertreiben, da keine Erlaubnis von Microsoft oder der Erwerb kostenpflichtiger Dokumentation dafür notwendig ist. Microsoft ist Marktführer, weil die Abwärtskompatibilität der
Windows-Plattform lange Zeit höchste Priorität bei der Weiterentwicklung von Windows besessen hat. Dies hat den Vorteil, dass Anwendungen nicht für jede Variante von Windows (vergangene oder zukünftige) angepasst werden müssen,
sondern der Plattform-Anbieter für die Kompatibilität verantwortlich ist.
Microsoft hat deswegen eine stetig wachsende Menge Shims, anwendungsspezifischer Work-arounds, in jede neue Windowsversion integrieren müssen.
Für den Nutzer ergibt sich der Vorteil, dass er seine Software in der gewünschten Version auch nach einem Windows-Upgrade weiterverwenden kann und er keine Änderungen vornehmen muss. Nachteil dieser stabilen aber proprietären
Windows-Plattform, bei denen keine Änderungen vorgenommen werden dürfen, ist der Lock-in-Effekt, durch den Betriebssystemwechsel werden aufgrund mangelnderInteroperabilität unattracktiv. Die Familie der Windows-Betriebssysteme
besitzt die Marktführerschaft auf dem PC-Markt mit einem Anteil von 80 bis 90 % laut verschiedener WebStat-Counter, gefolgt von Apple-Betriebssystemen.
